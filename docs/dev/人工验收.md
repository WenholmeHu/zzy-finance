# 人工验收

文档用途：指导当前版本的抖音对账功能做人工验收。  
适用范围：`聚天下 + 抖音` 对账流程。  
推荐验收目录：`D:\DaMoXing\ZZY\finance\.worktrees\douyin-reconciliation`

## 1. 验收前准备

### 1.1 进入项目目录

```powershell
cd D:\DaMoXing\ZZY\finance\.worktrees\douyin-reconciliation
```

### 1.2 安装依赖

如果本机已经装过依赖，可以跳过。

```powershell
pip install -r requirements.txt
```

### 1.3 确认样例文件存在

本次验收直接使用仓库内样例：

- `D:\DaMoXing\ZZY\finance\.worktrees\douyin-reconciliation\test_data\jutianxia.xlsx`
- `D:\DaMoXing\ZZY\finance\.worktrees\douyin-reconciliation\test_data\douyin.xlsx`

## 2. 启动服务

在项目根目录执行：

```powershell
uvicorn app.main:app --reload
```

浏览器访问：

```text
http://127.0.0.1:8000
```

预期结果：

- 页面可以正常打开
- 页面能看到 `对账月份`
- 页面能看到 `平台选择`
- 页面能看到两个上传框
- 页面能看到 `开始对账` 按钮

## 3. 页面提交流程

在页面中按下面方式操作：

1. 选择 `对账月份 = 2026-03`
2. 平台选择 `抖音`
3. 上传聚天下文件：`test_data\jutianxia.xlsx`
4. 上传外部平台文件：`test_data\douyin.xlsx`
5. 点击 `开始对账`

预期结果：

- 页面显示结果总览
- 页面显示主结果表
- 页面显示订单差异检查
- 页面显示 `导出 Excel`
- 页面显示 `导出差异表`

## 4. 结果总览核对

提交成功后，先核对摘要卡片：

- `对账月份` 显示为 `2026-03`
- `平台` 显示为 `抖音`
- `成功匹配订单数` 显示为 `1838`
- `汇总产品数` 显示为 `6`
- `被过滤的非当月平台记录数` 显示为 `9`
- `聚天下有、第三方当月无` 显示为 `0`
- `第三方有、聚天下当月无` 显示为 `1`

### 5.1 表头核对

主结果表应按下面顺序展示：

1. `产品名称`
2. `核销人次`
3. `销售额`
4. `技术服务费`
5. `佣金`
6. `服务商佣金`
7. `结算实付`
8. `采购金额`
9. `利润`

`益行远方文旅体惠民年卡(太湖源店)`

- `销售额 = 48.90`

- `佣金 = -4.09`

## 6. 订单差异检查核对

### 6.1 页面数量核对

- `聚天下有、第三方当月无` 数量应为 `0`
- `第三方有、聚天下当月无` 数量应为 `1`

### 6.2 页面表头核对

展开差异区后检查：

- 左侧差异表第二列表头是 `渠道订单号`
- 右侧差异表第二列表头是 `订单编号`

### 6.3 页面内容核对

- `聚天下有、第三方当月无` 区域应显示 `无`
- `第三方有、聚天下当月无` 区域应包含订单号 `1091502369578909793`

## 7. 导出 Excel 核对

### 7.1 主结果导出

点击 `导出 Excel` 后，检查下载文件：

- 文件名应为 `reconciliation_douyin_2026-03.xlsx`

打开文件后检查：

- 工作表名称为 `对账结果`
- `A1 = 对账月份`
- `B1 = 2026-03`
- `A2 = 平台`
- `B2 = 抖音`
- 第 4 行为主结果表表头
- 表头顺序与页面一致

### 7.2 差异表导出

点击 `导出差异表` 后，检查下载文件：

- 文件名应为 `difference_orders_douyin_2026-03.xlsx`

打开文件后检查：

- 存在两个工作表：
  - `聚天下有、第三方当月无`
  - `第三方有、聚天下当月无`
- 两个工作表中：
  - `A1 = 对账月份`
  - `B1 = 2026-03`
  - `A2 = 平台`
  - `B2 = 抖音`
  - `A4 = 序号`
- `聚天下有、第三方当月无` 工作表：
  - `B4 = 渠道订单号`
  - 因为当前无差异，`B5 = 无`
- `第三方有、聚天下当月无` 工作表：
  - `B4 = 订单编号`
  - `B5 = 1091502369578909793`

## 8. 异常提示抽查

可额外做一次简单异常验证：

1. 平台仍选择 `抖音`
2. 聚天下文件上传正确文件
3. 外部平台文件故意上传一个错误文件，或上传缺少目标工作表的 Excel
4. 点击 `开始对账`

预期结果：

- 页面不应白屏
- 页面顶部应显示错误提示
- 错误信息应尽量能看出是“工作表缺失”或“字段不符合要求”

## 9. 验收结论

如果第 3 到第 7 节全部符合预期，可认为本次 `聚天下 + 抖音` 对账功能已经达到可验收状态。  
如果第 8 节异常抽查也符合预期，说明基本的错误提示链路也可用。
