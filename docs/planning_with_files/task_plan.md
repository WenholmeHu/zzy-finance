## 关键问题
1. 本轮主交付物必须是 `项目分阶段开发流程.md`
2. 文档要面向 Codex，但人也要能快速看懂
3. 用户明确要求本轮不要使用 phased 技能
4. 当前正式开发从阶段 1 开始，不跨到完整业务实现

## 已做决策
| 决策 | 理由 |
|------|------|
| 仅使用 planning-with-files-zh 管理本轮规划 | 用户明确要求不使用 phased 技能 |
| 开发流程按“4 个阶段，建议 1-2 次开发会话”组织 | 既符合项目复杂度，也符合用户对开发节奏的判断 |
| 文档内容聚焦阶段目标、边界、验证、文件范围 | 便于后续 Codex 直接照文档推进 |
| 第一轮正式开发只落阶段 1：项目骨架与运行闭环 | 控制上下文和风险，先拿到可运行基线 |
