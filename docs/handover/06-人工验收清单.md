# 人工验收清单

## 1. 适用场景

这份清单适用于：

- 新接入一个平台之后
- 修改平台适配器之后
- 修改共享领域规则之后
- 准备把系统继续交给业务使用之前

## 2. 验收前准备

1. 确认本地服务可以正常启动
2. 确认至少跑过一次 `pytest tests -q`
3. 准备聚天下样例和目标平台样例
4. 明确本次验收月份与业务口径

## 3. 通用人工验收项

### 3.1 页面流程

- 首页可以正常打开
- 平台下拉框显示正确
- 月份选择与文件上传正常
- 提交后页面不会报错

### 3.2 对账结果

- 主结果表能正常显示
- 主结果表列顺序与平台口径一致
- 匹配成功订单数量符合预期
- 内部单边差异、外部单边差异符合预期

### 3.3 Excel 导出

- 主结果 Excel 可以导出
- 差异结果 Excel 可以导出
- 导出后的列标题与页面一致
- 导出文件中的平台名称和月份正确

## 4. 平台规则验收项

每次验收至少核对以下内容：

1. 月份过滤字段是否正确
2. 聚天下匹配字段是否正确
3. 平台订单号字段是否正确
4. 一单多行是否先合并再汇总
5. 销售额 / 结算实付 / 采购金额是否按平台规则映射
6. 利润是否等于 `结算实付 - 采购金额`

## 5. 建议至少挑 4 类订单核对

1. 正常匹配成功订单
2. 内部有、外部没有的订单
3. 外部有、内部没有的订单
4. 月份边界订单

如果平台存在退款、负数行或冲正行，还应额外抽样核对。

## 6. 当前默认可直接做的人工验收

仓库当前默认样例是同程，因此最方便的人工验收路径是：

1. 启动服务
2. 选择平台 `同程`
3. 选择样例对应月份
4. 上传：
   - `test_data/jutianxia_tc.xlsx`
   - `test_data/tongcheng.xlsx`
5. 检查页面结果和导出

## 7. 验收结论建议记录什么

建议至少记录：

- 验收日期
- 验收人
- 平台
- 对账月份
- 使用的样例文件
- 是否通过
- 发现的问题
- 是否已回归验证
