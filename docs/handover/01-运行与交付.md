# 运行与交付

## 1. 运行环境

- `Python 3.10+`
- Windows + PowerShell 已验证

项目当前是本地运行型工具，没有 Docker、Windows 安装包或服务化部署脚本。

## 2. 本地启动

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn app.main:app --reload
```

浏览器访问：

```text
http://127.0.0.1:8000
```

## 3. 业务使用方式

当前最稳定的使用方式是：

1. 启动本地服务
2. 在网页中选择平台与对账月份
3. 上传聚天下 Excel 和目标平台 Excel
4. 查看主结果与差异结果
5. 导出主结果 Excel 和差异订单 Excel

当前系统更接近“内部本地工具”，不是集中部署的多人在线系统。

## 4. 修改代码前建议先做的事

1. 跑一次 `pytest tests -q`
2. 看 [00-交接总览.md](./00-交接总览.md)
3. 看 [平台拓展开发说明.md](../dev/平台拓展开发说明.md)
4. 看目标平台真实样例

## 5. 修改代码后的最小回归

至少执行：

```powershell
pytest tests -q
```

如果只改某个平台，建议再补跑该平台相关测试，例如：

```powershell
pytest tests/platforms/test_tongcheng_adapter.py -q
pytest tests/integration/test_tongcheng_reconciliation.py -q
pytest tests/web/test_reconcile_route.py -q
```

## 6. 交付给公司继续使用时的建议

如果项目由后续同事继续日常使用，建议一并交付以下内容：

- 一份可直接启动的 Python 环境说明
- 当前可用的真实样例文件
- 最新的人工验收结果
- 本目录下的交接文档

如果未来要降低人工启动成本，建议补一个 `start.ps1` 或者公司内部统一的启动脚本。

## 7. 当前默认验证样例

仓库当前默认样例是同程：

- `test_data/jutianxia_tc.xlsx`
- `test_data/tongcheng.xlsx`

因此最方便的启动后人工验证路径，也是“聚天下 + 同程”。
